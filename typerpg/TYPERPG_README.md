# TypeRPG

타이핑 기반 실시간 턴제 RPG. 협동 가능. 댕청이 라인업의 영어 학습 + 게임 라인업의 한 축.

> **현재 단계**: Phase 5.8 (클래스 마스터리) — 같은 클래스로 던전 5회 클리어 시 **마스터** 등급. 패시브 효과 약 1.5배 강화 (Knight HP +50→+75, Mage 약점 +25%→+40%, Rogue 첫 *2* 타격 크리 등). 카운트는 자동, 시각적으로 ⭐⭐ 표시. 캐릭터 잠금 해제(Phase 5.7)와 결합해 *반복 플레이의 두 층위* 완성: 짧은 사이클(에센스로 새 캐릭터) + 긴 사이클(클리어로 마스터).

## 게임 컨셉 (3줄)

- 적이 공격을 차징하는 동안, 플레이어는 스킬 단어를 영문으로 타이핑해서 발동
- 각 캐릭터는 4개의 스킬 슬롯, 각 슬롯에 영단어 1개씩
- 캠페인(스토리·솔로) + 던전(로그라이크·솔로/코옵). 멀티는 던전에서만

## 8 클래스 — 어휘 카테고리 매핑

각 클래스는 한 어휘 카테고리를 담당. 클래스 선택 = 학습할 어휘 영역 선택이 되도록 설계.

| 클래스 | 한국어 | 테마 | 어휘 카테고리 | 스타터 스킬 4개 |
|---|---|---|---|---|
| `mage` | 마법사 | 원소 마법 | 자연 명사 | fire / ice / wave / wind |
| `archer` | 궁수 | 활과 화살 | 동작 명사+동사 | arrow / aim / shot / eye |
| `knight` | 기사 | 검과 방패 | 전투 명사+동사 | sword / shield / cut / guard |
| `rogue` | 도적 | 빠른 움직임 | 동작 동사 | slash / dash / hide / claw |
| `priest` | 사제 | 치유와 축복 | 치유/감정 | heal / bless / save / pray |
| `druid` | 드루이드 | 자연의 힘 | 자연/식물 | plant / leaf / thorn / tree |
| `paladin` | 성기사 | 신성한 빛 | 추상/감정 | holy / light / smite / faith |
| `monk` | 수도승 | 맨손 무술 | 신체 부위+동작 | punch / kick / bite / block |

> 3학년 영어 교육과정 어휘 분류(자연·신체·동작·감정 등)와 자연스럽게 매핑됨.

## 모드 구조

### 캠페인 (싱글, 스토리)
- 스테이지 진행형. 클래스를 점진적으로 해금
- **B → A 전환**: 초기 스테이지는 적 차징 없음(연습 모드 = B안). 진행하면서 차징 빠른 적이 섞임(= A안). 적 종류 자체로 난이도 차별화. *스테이지 번호로 끊지 않음.*
- 튜토리얼: ① 적 없음·타이핑만 → ② 허수아비 → ③ 슬라임(약한 반격) → ④ 차징 적 도입

### 던전 (싱글 또는 2인 코옵, 로그라이크)
- A안만 (차징 압박)
- 한 던전 5~10층, 10~15분 안에 끝나도록
- 메타 프로그레션 있음 (영구 화폐 → 강화)
- 1회 클리어 시 새 스킬 1개 영구 컬렉션에 추가

## 스킬 시스템

### 스타터 스킬 (영구, 4개)
- 클래스 선택 시 자동 부여
- 절대 사라지지 않음
- 캠페인만 해도 충분히 플레이 가능

### 던전에서 발견하는 레어 스킬
- 더 강하거나 특수 효과 (예: `fire` → `firestorm`)
- 던전 중에는 슬롯 덮어쓰기 (Hades 부운 방식)
- 던전 클리어 시 1개를 영구 컬렉션에 저장
- 로비에서 4 슬롯 자유 편성

### 희귀도 4단계
- Common (흰색) — 스타터
- Rare (파랑) — 잡몹 드롭
- Epic (보라) — 보스 드롭
- Legendary (금색) — 던전 최종 보스 드롭

### 클래스 제한 (제안, 미확정)
- 70% 본인 클래스 스킬 + 30% 와일드카드 (어느 클래스든 사용 가능)
- 클래스 정체성은 유지하되 약간의 자유도 부여

## Phase 3.5 — 원소 상성 & 상태이상

전투 시스템에 깊이가 들어옴. *수치 보정*이 아닌 *스킬이 하는 일 자체*로 클래스를 차별화.

### 5속성 체계

```
   🔥 fire ──→ 🌪️ wind ──→ 💧 water ──→ 🔥 fire   (3원소 순환)
                ✨ light ↔ 🌑 dark    (상호 약점)
```

**상성 배율**:
- 약점 적중: **1.5x** + 금색 큰 데미지 + "GOOD!" 텍스트
- 같은 속성 / cycle 반대 / light·light / dark·dark: **0.5x** + 회색 작은 데미지 + "RESIST" 텍스트
- 그 외 (무속성 포함): **1.0x**

### 발견 학습 원칙

**적의 속성만 표시, 약점/저항은 절대 안 보임**. HP 바 왼쪽에 적 속성 badge (예: 🌪️ 바람) 하나만. 플레이어가 직접 cycle을 떠올려서 스킬 선택. 잘 들어가면 "GOOD!" 피드백으로 학습 강화. 잘 안 들어가면 "RESIST" 피드백으로 *틀린 매칭임을 기억*. 튜토리얼 없음 — *시행착오로 체득*.

### 적 속성 분포 (22종)

| 속성 | 적 |
|---|---|
| 🔥 fire   | dragon, flamevil, candlesnake, baby_dragon |
| 💧 water  | frog, fish |
| 🌪️ wind   | bamboo, electric_bat, bird |
| 🌑 dark   | centipede, hornheart, mushroom, mimic |
| ✨ light  | (없음 — v1에선 dark 약점만 활용) |
| ⚪ 무속성 | golem, goblinboss, goblinball, radish_plant, mutant, mole, monkey, racoon, golem_guard |

### Mage 스킬 셋 변경

기존 `fire / ice / wave / wind` → **`fire / wave / wind / light`**

(ice는 cycle에 없어서 light로 교체. Mage가 5개 속성 중 4개 커버, dark는 advanced 영역으로 유보)

### 클래스별 스킬 동작 분화

| 클래스 | 액션 유형 분포 |
|---|---|
| **Mage**   | 4 데미지 (모두 원소 부여). `fire`에 40% 확률 burn, `wind`에 100% freeze 부수효과 |
| **Knight** | 2 데미지(`sword`/`cut`) + `shield` 보호막 30 HP + `guard` -40% 피해 감소 8초 |
| **Priest** | 2 자가 회복(`heal`+25, `bless`+15) + 2 데미지(`save`/`pray`) |
| **나머지 5 클래스** | 4 데미지 (무속성, 안정적 1.0x) |

### 방어 시스템 — 두 종류 명확히 분리

**🛡 보호막 (Barrier)** — *HP-기반 흡수, 시간 무제한, 누적 안 됨*
- Knight `shield` 스킬 → 30 HP의 보호막 생성. 받는 피해를 *먼저* 흡수
- 이미 보호막이 남아있어도 새로 걸면 *기존 것 덮어쓰기* (5 + 30 = 30, NOT 35)
- HP 바 위에 청록색 별도 bar로 표시 (`🛡 30/30`)
- 보호막이 깨지면 "BREAK!" 텍스트
- 쿨다운 **8초** — 한 번 쓰면 한참 못 씀, 신중한 사용 강요

**🛡 방어 자세 (Guard)** — *% 데미지 감소 버프, 시간 한정*
- Knight `guard` 스킬 → 8초간 받는 피해 40% 감소
- 상태 아이콘으로 표시 (남은 시간 카운트다운)
- 보호막과 *동시 적용 가능* — guard로 데미지 줄이고 → 보호막이 줄어든 양만 흡수
- 쿨다운 **10초** — 8초 지속 + 2초 노출 구간

**데미지 흐름** (적이 18 데미지로 공격할 때):
```
1. 18 데미지 발생
2. Guard 활성 → 18 × 0.6 = 11 데미지로 감소
3. 보호막 12 HP 남음 → 11 모두 흡수, 보호막 1 HP 남음
4. HP는 100 → 100 (안 깎임)
```

### 스킬별 쿨다운

| 스킬 | CD | 의도 |
|---|---|---|
| 공격 스킬 (기본) | 1.5s | 빠른 연타 |
| Mage `wind` (freeze 100%) | **2.5s** | 너무 강력해서 약간 페널티 |
| Priest `bless` (+15 HP) | **3s** | 짧은 회복은 비교적 자주 |
| Priest `heal` (+25 HP) | **5s** | 큰 회복은 신중하게 |
| Knight `shield` (보호막) | **8s** | 한 번 걸고 공격 난사 |
| Knight `guard` (방어 자세) | **10s** | 가장 긴 CD, 가장 긴 효과 |

**핵심 의도**: *짧은 CD 공격 스킬은 난사용, 긴 CD 버프는 한 번씩 걸고 공격 사이클로 돌아가는* 리듬. 보스전이 단조로운 "패턴 외우기"가 아니라 *"방어 타이밍 결정 → 공격 윈도우 만들기"* 가 됨.

### 크리티컬 시스템

- 모든 공격에 **10%** 기본 크리티컬 확률
- 크리티컬 시 **1.5x 데미지** + 큰 핑크색 데미지 숫자 (회전 효과) + "**CRIT!**" 스테이지 중앙 banner
- 원소 약점과 중첩 가능: weakness(1.5x) × crit(1.5x) = 2.25x 최대 (드물지만 큰 한 방)
- Archer 패시브로 크리티컬 확률 +15% (총 25%)
- Rogue 패시브로 새 적의 첫 타격은 무조건 크리

### 콤보 시스템

같은 적에게 연속 적중하면 데미지 보너스. 6초 안에 다음 적중 없거나 적이 죽으면 리셋.

| 콤보 | 보너스 |
|---|---|
| 3히트 | +10% |
| 5히트 | +20% |
| 7히트 | +30% |
| 10히트+ | +50% |

시각: 좌상단 금색 카운터 ("**x5 COMBO +20%**") + 임계점(3/5/7/10)에서 큰 banner 텍스트. 10콤보는 빨간색 + 더 큰 폰트로 강조.

### 양방향 상태이상

이전엔 단방향(플레이어→적)이었는데 이제 **적도 플레이어에게** 상태이상 부여:

| 적 속성 | 부여 상태이상 | 확률 |
|---|---|---|
| 🔥 fire 보스 (dragon, flamevil) | burn (3초간 매초 5 데미지) | 40% |
| 🔥 fire 잡몹 (candlesnake, baby_dragon) | burn | 25~30% |
| 🌑 dark 보스 (centipede, hornheart) | **weaken** (4초간 자기 데미지 -30%) | 40% |
| 🌑 dark 잡몹 (mushroom, mimic) | weaken | 30% |
| 그 외 | 없음 (순수 데미지) | — |

플레이어 burn 처리: tick()에서 매초 5 데미지 적용, 영웅 카드에 🔥 상태 아이콘 + 카운트다운. weaken 처리: 데미지 계산에서 outgoing damage × 0.7. 빨간 화살표(malus.png) 아이콘 사용.

### 디버프 — Rogue의 통제 스킬

Rogue가 단순 DPS에서 *통제 캐릭터*로 격상:

- `dash` (질주) → **stall**: 적의 현재 차징에 즉시 +3초 추가. CD 6초
- `hide` (숨다) → **dodge**: 다음 적 공격 1회 회피. 적용 시 영웅 카드에 회피 태세 아이콘. 적 공격 시 "**MISS!**" 노랑 텍스트와 함께 소비. CD 8초

`slash`/`claw`는 일반 데미지 유지. 이제 Rogue는 *"dash로 적 공격 늦추고 → hide로 다음 공격 무효 → 그 사이 slash/claw 난사"* 흐름.

### 8 클래스 패시브 (자동 적용)

각 클래스 카드와 영웅 카드에 ⭐ 별표 라벨로 표시됨.

| 클래스 | 패시브 | 효과 |
|---|---|---|
| **Mage** | ⭐ 원소 통달 | 약점 적중 시 데미지 +25% (1.5x → 1.75x) |
| **Archer** | ⭐ 정확한 사격 | 크리티컬 확률 +15% (10% → 25%) |
| **Knight** | ⭐ 강철 의지 | 최대 HP +50 (100 → 150) |
| **Rogue** | ⭐ 기습 | 새 적의 첫 공격은 100% 크리티컬 |
| **Priest** | ⭐ 축복받은 자 | HP ≤30%일 때 매초 5 자동 회복 |
| **Druid** | ⭐ 자연의 가호 | 5초마다 5 HP 자동 회복 (지속) |
| **Paladin** | ⭐ 성스러운 방패 | 받는 모든 피해 -10% |
| **Monk** | ⭐ 빠른 손 | 모든 스킬 쿨다운 -20% |

이제 클래스가 진짜 *역할*을 가짐:
- **Mage** = 약점 저격 폭딜
- **Archer** = 크리 노리는 DPS
- **Knight** = HP 폭증 탱커
- **Rogue** = 통제 + 첫 타격 폭딜
- **Priest** = 위급시 자동 회복 (HP 30% 진입 시 자가 보호)
- **Druid** = 지속 회복 누적
- **Paladin** = 안정적 광역 감쇠
- **Monk** = CD 단축으로 공격 횟수 +25% (1/0.8 = 1.25)

### 확장 시스템 검증

7개 시스템 자동 검증 통과:

```
Knight 최대 HP                  → 150 ✓
Rogue dash → 적 차징 +3초      → 9.5s → 12.1s ✓
Rogue hide → 회피 적용/소비    → HP 100→100, 자동 제거 ✓
Rogue 첫 타격 크리             → slash 15 → 23 (1.5x) ✓
콤보 5 누적                    → 5/5 ✓
Dragon 공격 → burn 부여        → player 상태이상 활성 ✓
플레이어 burn 매초 5 데미지    → tick()에서 적용 ✓
```

### 전체 동작 검증

**원소 매트릭스 (4 mage 스킬 × 5 적 속성 = 20 조합)**: 모두 정확한 배율로 데미지 적용 ✓

```
fire   vs fire/water/wind/dark/none = 0.5/0.5/1.5/1.0/1.0  ✓
wave   vs fire/water/wind/dark/none = 1.5/0.5/0.5/1.0/1.0  ✓
wind   vs fire/water/wind/dark/none = 0.5/1.5/0.5/1.0/1.0  ✓
light  vs fire/water/wind/dark/none = 1.0/1.0/1.0/1.5/1.0  ✓
```

**8 클래스 핵심 스킬 동작** (Phase 3에서 검증한 데미지 외 추가):
- Mage `fire` 발동 → burn 상태 확률 부여 ✓
- Mage `wind` 발동 → freeze 상태 부여, 차징 정지 ✓
- Priest `heal` 발동 → +25 HP 즉시 회복, 초록 +25 텍스트, 영웅 카드 초록 글로우 ✓
- Knight `shield` 발동 → shield 상태 3초, 영웅 카드 파란 글로우, 받는 피해 50% 감소 ✓

- light 속성 적 추가 — 현재 없음 (mage `light`는 dark 적 대상으로만 의미 있음)
- 상태이상 추가 (poison, stun, regen-over-time 등) — 현재 3종만
- 적이 플레이어에게 상태이상 부여 — 현재 단방향
- 적 속성 시각 효과 (예: fire 적에게 빨간 오라) — badge로만 표시

## Phase 4 — 캠페인 스테이지

10 스테이지 순차 진행 + 메인 메뉴 + 진행도 저장(localStorage). 모든 이전 시스템이 캠페인 흐름 속에서 자연스럽게 활성화됨.

### 화면 흐름

```
페이지 로드 → 메인 메뉴
              ├─ 캠페인 시작 → 클래스 선택 → Stage 1 → 클리어 → Stage 2 자동 → ... → Stage 10 → 엔딩
              ├─ 이어하기 (Stage N부터) — 진행도가 있을 때만 활성화
              ├─ 자유 모드 → 클래스 선택 → 자유 전투 (이전 phase 동작 그대로)
              └─ 진행도 초기화
```

배틀 화면 좌상단 "← 메인" 버튼: 캠페인 중엔 메인 메뉴로, 자유 모드에선 클래스 선택으로.

### 10 스테이지 — 의도된 학습 곡선

각 스테이지가 *하나의 새 시스템*을 자연스럽게 가르치도록 큐레이션:

| # | 이름 | 적 (속성) | HP | ATK | 차징 | 학습 포인트 |
|---|---|---|---|---|---|---|
| 1 | 준비 운동 | 무 식물 (무) | 30 | 0 | ∞ | *적 공격 없음* — 타이핑 사이클만 익힘 |
| 2 | 첫 위협 | 원숭이 (무) | 50 | 8 | 12s | *차징* 도입 — 적도 공격함을 학습 |
| 3 | 바람의 적 | 새 (🌪️ 바람) | 55 | 10 | 10s | *원소 약점 발견* — fire가 GOOD! |
| 4 | 물의 적 | 개구리 (💧 물) | 60 | 11 | 10s | *상성 변화* — 이전 약점이 안 통함 |
| 5 | 첫 보스 | 고블린 왕 (무) | 100 | 15 | 9s | *보스 압박* — 콤보/크리 활용 |
| 6 | 어둠 속에서 | 버섯 (🌑 어둠) | 75 | 12 | 9s | *light 활용* + weaken 부여 경험 |
| 7 | 불꽃의 습격 | 촛불뱀 (🔥 불) | 120 | 16 | 9s | *양방향 burn* — 적이 화상 부여 |
| 8 | 뿔심장 | 뿔심장 (🌑 어둠) | 140 | 16 | 10s | weaken 보스 — 방어 스킬 필수 |
| 9 | 불의 악마 | 불의 악마 (🔥 불) | 170 | 18 | 9s | 모든 시스템 통합 — burn + 약점 + 콤보 |
| 10 | 최종: 드래곤 | 드래곤 (🔥 불) | 220 | 20 | 8s | **최종 보스** — 그동안 배운 모든 것 |

각 스테이지는 자체 배경(`bg`/`sky`/`floor`/`props`) override로 분위기 차별화. Stage 1의 벚꽃 → Stage 7~10의 협곡 + 불타는 하늘로 시각적 점진 전환.

### 스테이지 진행 UI

- **상단 중앙 인디케이터**: `STAGE 3 / 10 · 바람의 적` (금색 테두리 + 글로우)
- **튜토리얼 말풍선**: 스테이지 진입 시 5초간 표시 후 자동 사라짐. 적 위에 화살표 달린 노란 말풍선
- **승리 자동 진행**: 클리어 후 1.8초 딜레이 → 다음 스테이지 자동 진입 + 새 튜토리얼
- **패배 재시도**: 같은 스테이지로 재시작 (override 유지)
- **localStorage 저장**: `typerpg_max_stage` 키로 가장 높은 클리어 스테이지 저장. 새로고침해도 유지

### 메인 메뉴 화면

- v0.4 · Phase 4 Campaign 푸터

## Phase 5 — 던전 & 레어 스킬 발견

캠페인은 *한 번 클리어 후 끝*인 학습 경로였다면, 던전은 *반복 플레이 가능한 본 게임 루프*. 매 던전마다 무작위 적 시퀀스 + 보상 선택 + 새 영단어 발견의 두근거림.

### 던전 구조

```
메인 메뉴 → 던전 선택 → 클래스 선택 → 룸 1 → 보상 → 룸 2 → 보상 → ... → 룸 6 → 보상 → 룸 7 (보스) → 클리어!
```

- **7 룸**: 1~6번은 mob 풀에서 무작위 (즉시 중복 회피), 7번은 boss 풀에서 무작위
- **난이도 곡선**: 룸 1~3는 평이, 4~5는 강화 (HP ×1.2~1.3, ATK ×1.1~1.15), 룸 6은 엘리트 (×1.4), 룸 7은 보스 (기본 보스 스탯 그대로)
- **배경 자동 매칭**: 적의 속성에 따라 bg/sky/floor/props가 자동 결정 (fire 적 → canyon + 불타는 하늘, dark 적 → 어두운 대나무 + 별이 빛나는 밤)
- **한 세션 ~5~10분**: 짧고 명확한 사이클이 반복 플레이 동기를 만듦
- **사망 시 진행 리셋**: 다시 룸 1부터. 발견한 스킬도 사라짐 (한 던전 내에서만 유효)

### 보상 시스템 (매 룸 클리어 시, 보스 제외)

3개 카드 중 1개 선택:

| 카드 | 효과 | 지속 |
|---|---|---|
| 🩹 **HP 회복** (초록) | HP +30 즉시 | 즉시 |
| ⚔️ **단련** (주황) | 다음 룸 데미지 +30% | 1 룸 한정 (다음 보상에서 1.0으로 리셋) |
| ✨ **새 스킬 발견!** (보라) | 레어 스킬 1개 → 5번째 슬롯 추가 | 한 던전 전체 |

발견 카드는 한 던전당 최대 1번만 제시됨 (이미 스킬 받았으면 풀에서 제외). 항상 회복 + 단련은 제공되어 *최소한의 안전망*은 보장.

### 레어 스킬 풀 (12종)

매번 무작위로 1개 발견. 모두 3학년 수준 영단어 + 원소 시스템 통합:

| 영단어 | 한글 | 속성 | 액션 | 특수 |
|---|---|---|---|---|
| **blaze** | 맹화 | 🔥 fire | damage | burn 60% 확률 |
| **flame** | 불꽃 | 🔥 fire | damage | — |
| **flood** | 홍수 | 💧 water | stall | 적 차징 +2초 |
| **tide** | 조류 | 💧 water | barrier | HP 20 보호막 |
| **mist** | 안개 | 💧 water | damage | — |
| **gust** | 돌풍 | 🌪️ wind | damage | freeze 100% (CD 2.5s) |
| **glow** | 광휘 | ✨ light | heal | HP +15 |
| **void** | 공허 | 🌑 dark | damage | — |
| **boom** | 폭발 | ⚪ none | damage | — |
| **gem** | 보석 | ⚪ none | heal | HP +20 |
| **iron** | 강철 | ⚪ none | guard | -40% 받는 피해 8초 |
| **swift** | 신속 | ⚪ none | dodge | 다음 공격 회피 |

각 클래스에 이미 있는 스킬은 자동 필터링. 예: Mage가 `fire`를 가지고 있으니 `flame` (다른 단어) 같은 비중복만 풀에서 추출.

### UI 통합

- **메인 메뉴**: 보라 "던전 (로그라이크)" 버튼 + 누적 클리어 횟수 표시 (`🏆 던전 클리어: 3회`)
- **던전 인디케이터**: 상단 중앙 보라 테두리 `DUNGEON 4 / 7 · 룸 4` (보스 룸은 `⚠️ 보스`)
- **5번째 스킬 슬롯**: 발견 시 영웅 카드에 자동 추가
- **보상 화면**: 풀스크린 오버레이, 3 카드 가로 배치, hover 시 색상별 글로우 + 떠오름
- **던전 클리어 화면**: 트로피 🏆 + DUNGEON CLEAR! + 통계 (처치 수, 클래스, 발견 스킬, 누적 클리어) + 2 버튼 (다시 던전 / 메인 메뉴)

### 메타 프로그레션 (v1 단순화)

- `typerpg_dungeon_clears` localStorage 키에 누적 카운트만 저장
- 새로고침해도 유지
- 메인 메뉴에서 표시 ("🏆 던전 클리어: 3회")
- 영구 업그레이드 (시작 HP +10, 크리 +5% 등)는 **Phase 5.5 이후**로 유보

### 검증 결과

자동 테스트 통과:

```
던전 시작                → 룸 1, 적 시퀀스 7개 생성 ✓
룸 1 클리어              → 보상 화면 3 카드 표시 ✓
discover 선택            → state.dungeonExtraSkill = 'void' (예시) ✓
룸 2 진입                → 5번째 슬롯 영웅 카드에 등장 ✓
6 룸 + 보상 사이클       → 룸 7 = 무작위 보스 (예: hornheart) ✓
보스 처치                → 던전 클리어 화면 표시 ✓
localStorage 클리어 저장 → 누적 횟수 +1 ✓
사망 시 던전 상태 리셋   → dungeonRoom/extraSkill/sequence 초기화 ✓
```

### 의도적으로 미구현 (Phase 5.5+ / Phase 6+)

- **영구 업그레이드 시스템** (메타 통화로 시작 HP/크리 등 강화) — 게임 밸런스 더 다듬은 후
- **다층 던전** (1층 클리어 후 2층 진입, 누적 적용되는 강화) — 한 세션 너무 길어짐
- **엘리트 룸 선택지** (위험 ↔ 큰 보상) — 룸 4~6에 옵션으로
- **레어 스킬 풀 확장** — 현재 12종, 학년 별 어휘 진도 맞춰 확장
- **다중 레어 스킬 슬롯** (한 던전에 2~3개 발견) — 현재 1개만
- **이벤트 룸** (회복샘 / 상점 / 도전) — 비전투 컨텐츠
- **저장된 던전 이어하기** (중단해도 룸 N에서 재개) — 현재 사망=리셋

## Phase 5.5 — 메타 프로그레션 + 다층 + 이벤트 룸

Phase 5에서 *한 번의 던전 안에서만* 의미 있던 요소들을 *세션을 넘는 영구 진행감*으로 확장. 본격 게임 사이클 완성.

### 1) 영구 메타 업그레이드 (💎 에센스)

#### 통화 시스템
- **에센스**는 던전 클리어로 획득 (Floor별 차등):
  - Floor 1 클리어: **+3 에센스**
  - Floor 2 클리어: **+5 에센스**
  - Floor 3 클리어: **+8 에센스** (전체 완주 시 16 누적)
- localStorage `typerpg_meta`에 영구 저장 (essence + upgrades)
- 사망 시 *이번 던전에서 모은 미정산 에센스 손실* (이전 누적분은 안전)

#### 4종 강화 (5단계까지 누적)
| 아이콘 | 이름 | 효과 | 비용 | 최대 |
|---|---|---|---|---|
| ❤️ | 시작 HP | 단계마다 +10 HP | 3 에센스 | +50 HP (5단계) |
| 🛡️ | 시작 보호막 | 단계마다 +10 보호막 | 3 에센스 | +50 보호막 |
| ⚔️ | 데미지 강화 | 단계마다 +5% 데미지 | 5 에센스 | +25% |
| ✨ | 크리 확률 | 단계마다 +3% 크리 | 5 에센스 | +15% |

**모든 모드에 적용** (캠페인 + 던전 + 자유). 메인 메뉴의 **💎 강화 상점** 청록 버튼에서 구매.

### 2) 다층 던전 (3 Floors)

이제 던전은 **3층**까지 도달 가능. 각 층 = 7 룸 = 21 룸 총.

#### 층별 강화 multiplier
| Floor | HP × | ATK × | 클리어 보상 |
|---|---|---|---|
| F1 | 1.0 | 1.0 | +3 에센스 |
| F2 | 1.5 | 1.3 | +5 에센스 |
| F3 | 2.0 | 1.5 | +8 에센스 |

#### 층 전환 결정
보스 처치 후 Floor 클리어 화면:
- **▶ Floor N+1 진입** (주황) — 더 위험, 더 큰 보상
- **💰 지금 끝내기** (초록) — 안전하게 현재 누적분 확정

게임의 핵심 긴장감: *더 가서 큰 보상 노리느냐, 챙길 거 챙기고 빠지느냐.* 

레어 스킬은 *층을 넘어 유지*되고, 발견 슬롯도 누적 (Floor 1에서 1개 + Floor 2에서 1개 = 6 슬롯 영웅).

### 3) 이벤트 룸 (50% 확률 등장)

각 던전 Floor의 룸 3 또는 룸 5 위치에 *50% 확률로* 이벤트 룸이 배치됨:

#### 🌿 회복샘
- 자동 효과: HP 완전 회복 + 보호막 +30
- "🌿 회복샘 +HP +🛡" 초록 banner 표시
- 2초 후 자동으로 다음 룸 진입 (전투 없음)
- 인디케이터에 `🌿 회복샘` 라벨

#### ⚔️ 도전
- 강력한 적 (HP × 2.0, ATK × 1.4)
- 처치 시 *2개의 레어 스킬 중 선택* + "대신 회복 (HP +40)" 옵션
- 인디케이터에 `⚔️ 도전` 라벨
- 고위험 고보상 — 강력한 스킬을 *2개 옵션 중 골라* 가져갈 수 있는 유일한 기회

### 4) 다중 레어 스킬 발견 (최대 2개)

이전: 한 던전 = 최대 1개 발견. 이제: **최대 2개**.

#### 발견 빈도
- 발견 0개: 보상 화면에 discover 카드 *항상* 등장
- 발견 1개: discover 카드 *50% 확률* 등장
- 발견 2개: discover 카드 제외

#### UI
영웅 카드 스킬 그리드: **기본 4 + 발견 0~2 = 최대 6 슬롯**. 발견할 때마다 새 슬롯이 자동 등장.

### 검증 결과

모든 시스템 자동 테스트 통과:

```
메타 상점 표시               → 4 업그레이드 카드 + 에센스 표시 ✓
HP +10 구매                  → 30 에센스 → 27, level 1 ✓
던전 시작 (meta +10 적용)    → playerMaxHp = 110 ✓
이벤트 룸 자동 생성          → 룸 5 = spring (50% 확률) ✓
회복샘 자동 효과              → HP+보호막 복구, 자동 진행 ✓
Floor 1 클리어 화면          → +3 에센스 표시, 계속/끝내기 선택 ✓
Floor 2 진입 (multiplier)    → 적 HP × 1.5 적용 ✓
도전 룸 강화                  → 적 HP × 2.0 적용 ✓
도전 보상 (2 스킬 선택)      → 3 카드 (스킬2 + 회복1) 표시 ✓
다중 레어 스킬                → state.dungeonExtraSkills = ['tide', 'gem'] ✓
사망 시 미정산 에센스 손실   → state.dungeonEssenceEarned = 0 리셋 ✓
```

### 데이터 흐름 종합

```
던전 시작
  ├─ meta.essence (영구) 그대로
  ├─ meta.upgrades (영구) 적용 → 시작 HP/보호막/데미지/크리 강화
  └─ dungeonEssenceEarned = 0 (이번 런 누적)

Floor 클리어 (룸 7 보스 처치)
  ├─ dungeonEssenceEarned += FLOOR_MULTIPLIERS[floor].essenceReward
  ├─ 마지막 층이면: finalizeRunEssence() → meta.essence += dungeonEssenceEarned
  └─ 아니면: Floor 클리어 화면 → 계속/끝내기 선택
       ├─ 계속: 다음 층 진입 (dungeonEssenceEarned 유지, 미정산)
       └─ 끝내기: finalizeRunEssence() → 안전 정산

사망 (HP 0)
  └─ dungeonEssenceEarned = 0 (이번 런 손실) + 모든 dungeon state 리셋
```

### 의도적으로 미구현 (Phase 6+)

- **상점 룸** (전투 중 에센스 일부 소비 구매) — 룸 보상으로
- **추가 강화 종류** (콤보 시작 +N, CD 단축, 클래스별 강화)
- **저장된 던전 이어하기** (Floor 2 중간에 종료 → 다음 세션에 이어서)
- **챌린지 모드** (모든 강화 잠금 상태로 클리어 도전) — 어려운 모드
- **무한 던전** (Floor 4, 5, 6... 끝없이) — 점수 모드
- **에센스로 직접 구매하는 레어 스킬** (영구 잠금 해제)

## Phase 5.6 — 클래스 정체성 튜토리얼 + 사운드

### 문제 인식

이전 캠페인은 *플레이어가 어떤 클래스를 골랐든* 같은 튜토리얼 텍스트를 보여줬어. 그런데 6/10 스테이지가 *원소 약점 발견*에 초점이 맞춰져 있어서, 원소 시스템을 활용 못 하는 7개 클래스(mage 제외)에는 *어색하거나 잘못된 안내*가 됐어.

예: Knight가 Stage 6 (어둠 적, 약화 부여)에 도달했을 때:
- ❌ 이전 텍스트: "✨ 빛이 효과적입니다" → Knight에겐 *light 스킬이 없음*
- ✅ 새 텍스트: "약화 중에도 shield/guard로 안정적 방어 가능"

### 해결책: 클래스 적응형 튜토리얼

각 스테이지의 `tutorial`이 *string에서 `function(classId)`로 변환*. 같은 적이지만 *플레이어의 도구*에 맞춘 안내.

#### 핵심 클래스 정체성 힌트

```js
const CLASS_HINTS = {
  mage:    '마법사는 원소 약점을 노려 1.5배 데미지!',
  archer:  '궁수는 크리 확률 +15%, 연속 공격으로 콤보 노리기',
  knight:  '기사는 HP 150 탱커. shield/guard로 방어전',
  rogue:   '도적의 첫 타격은 자동 크리! dash로 적 늦추고 hide로 회피',
  priest:  '사제는 heal/bless로 HP 관리, HP 30% 미만 시 자동 회복',
  druid:   '드루이드는 5초마다 자동 회복, 장기전 강함',
  paladin: '성기사는 받는 피해 -10%, 안정적 페이스',
  monk:    '수도승은 모든 CD -20%, 빠른 연타 가능',
};
```

#### 스테이지별 적응 예시

**Stage 10 (드래곤 최종전)** — 8개 클래스 각각 다른 전략 안내:

| 클래스 | 튜토리얼 텍스트 |
|---|---|
| Mage | 🐲 최종! water + light + 콤보 + 크리, 모든 무기를 쏟아부으세요 |
| Knight | 🐲 최종! shield+guard 적극 사용, HP 150이 당신의 가장 큰 자산 |
| Priest | 🐲 최종! heal/bless 사이클 + HP 30% 미만 자동 회복 활용 |
| Rogue | 🐲 최종! 첫 타격 자동 크리 + dash로 보스 차징 늦추기 |
| Archer | 🐲 최종! 크리 확률 25%, 거의 절반은 크리 — 연타로 승부 |
| Druid | 🐲 최종! 5초마다 자동 회복, 장기전 페이스 유지 |
| Paladin | 🐲 최종! 받는 피해 -10% + 안정적 운영 |
| Monk | 🐲 최종! CD -20%, 다른 클래스보다 25% 더 자주 공격 |

각 클래스가 *자신의 강점에 집중하라*는 안내를 받음. mage만의 게임이 아니게 됨.

### 사운드 효과 (Web Audio API)

오디오 파일 없이 *프로그램으로 합성*. 짧은 8비트-스타일 비프음으로 핵심 이벤트에 청각 피드백:

| 함수 | 트리거 | 음 |
|---|---|---|
| `playHit()` | 모든 데미지 표시 | 180Hz 사각파 0.08s |
| `playCrit()` | 크리티컬 발동 | 90Hz 톱니파 + 720Hz 사각파 (이중) |
| `playHeal()` | HP 회복 | 440 → 660Hz 사인파 (상승) |
| `playBarrier()` | 보호막 발동 | 330Hz 삼각파 |
| `playCombo(n)` | 콤보 3/5/7/10 임계점 | 660/784/880/1047Hz (콤보 높을수록 고음) |
| `playVictory()` | 승리 | C5 → E5 → G5 (메이저 코드) |
| `playDefeat()` | 패배 | 440 → 330 → 220Hz (하강) |
| `playReward()` | 보상 화면 등장 | 523 → 784Hz |

#### 사운드 토글

메인 메뉴 푸터에 작은 토글 버튼 **🔊 사운드 ON** / **🔇 사운드 OFF**. 상태는 localStorage `typerpg_sound_off`에 저장. 교실 환경 등에서 켰다 껐다 자유롭게.

### 검증 결과

```
8 클래스 × 3 핵심 스테이지 = 24 텍스트 모두 다름        ✓
- Stage 3 (바람): mage="fire 1.5배" / knight="HP 150 탱커"...
- Stage 6 (약화): mage="light 효과적" / priest="heal 정상 작동"...
- Stage 10 (드래곤): 8개 모두 클래스별 다른 전략

Web Audio API ctx 초기화                                ✓
SOUND.enabled = true (localStorage 기본값)              ✓
사운드 토글 버튼 동작 + 저장                            ✓
Knight Stage 6에서 새 튜토리얼 표시 확인                ✓
  "🌑 약화 부여! 약화 중에도 shield/guard로 안정적 방어 가능"
```

### 의도적으로 미구현 (Phase 6+)

- **모바일 터치 입력** — 키보드 타이핑 의존, 모바일은 가상 키보드 사용 시 UX 검토 필요
- **추가 메타 강화 종류** (콤보 시작 +N, CD 단축 등) — 현재 4종으로도 충분히 깊음
- **던전 중간 저장** — 5~10분 세션은 한 번에 진행 가능
- **무한 던전 모드** (Floor 4+) — Floor 3까지로 한 세션 30분, 적정
- **음악/BGM** — 효과음만 있고 배경 음악 없음. 추가 시 라이선스 고려

## Phase 5.7 — 캐릭터 잠금 해제 + 반복 플레이 유도

### 설계 의도

이전 phases에서 *깊이*는 충분했지만 (캠페인 10 스테이지, 던전 3 층, 메타 강화 4종), **장기 플레이 동기**가 약했어. 한번 클리어하고 강화 다 사면 *더 할 것*이 없었음.

해결책: **8 클래스 중 2개만 기본 무료**, 나머지 6개는 **에센스로 잠금 해제**. 매 던전 클리어 → 에센스 적립 → 새 클래스 해제의 사이클이 *반복 플레이의 장기 목표*가 됨.

### 시작 캐릭터 (무료)

- **🧙 Mage**: 원소 마법사 — 캠페인의 핵심 학습 도구 (원소 상성)
- **⚔️ Knight**: 탱크 — 대조되는 플레이 스타일 (방어 + HP +50)

*두 클래스 = 게임의 핵심 메커니즘 두 축*: 공격 vs 방어, 원소 vs 무속성. 신규 플레이어가 게임의 양극을 모두 경험할 수 있도록.

### 해제 가능 (각 10💎)

| 캐릭터 | 패시브 | 특징 |
|---|---|---|
| 🏹 **Archer** | 정확한 사격 (크리 +15%) | 입문 DPS — 크리 빈도 높아 만족감 |
| ✨ **Priest** | 축복받은 자 (HP ≤30% 매초 5 회복) | 입문 서포터 — 자동 안전망 |
| 🗡️ **Rogue** | 기습 (첫 타격 자동 크리) | 통제 + 폭딜 |
| 🌿 **Druid** | 자연의 가호 (5초마다 +5 HP) | 지속 서스테인 |
| 🛡️ **Paladin** | 성스러운 방패 (받는 피해 -10%) | 안정형 탱커 |
| 🥋 **Monk** | 빠른 손 (모든 CD -20%) | 가속 어그레시브 |

총 **60💎** 모두 해제 = **F3 풀 완주 약 4번** (16💎 × 4 = 64). 영구 강화 ~75💎와 비슷한 규모.

### UI 변경

#### 메인 메뉴 → 강화 상점 (이제 *두 탭*)

```
┌──────────────────────────────┐
│ 💎 강화 상점        보유: 30  │
├──────────────────────────────┤
│  [💪 강화]    [👥 캐릭터]    │  ← 탭 전환
├──────────────────────────────┤
│  탭 내용...                   │
└──────────────────────────────┘
```

- **💪 강화 탭**: 기존 4종 영구 업그레이드 (시작 HP / 보호막 / 데미지 / 크리)
- **👥 캐릭터 탭**: 8 클래스 카드 표시
  - 기본 (Mage, Knight): 황금 테두리 + ⭐ "기본 캐릭터" 라벨
  - 해제됨: 초록 테두리 + "✓ 해제됨" 라벨
  - 잠김: 회색 + 💎 10 해제 버튼 (에센스 부족 시 비활성)

#### 클래스 선택 화면 (잠금 표시)

잠긴 클래스 카드는:
- 그레이스케일 (회색 처리) + 어둡게
- 🔒 큰 자물쇠 아이콘 중앙
- "잠김" 라벨 + "💎 10 에센스" 청록 박스 + "상점에서 해제" 안내
- 클릭 시 *흔들림 애니메이션* (`lock-shake`) — 거절 피드백

### 역호환 (기존 플레이어 보호)

`loadMeta()`가 기존 진행 데이터를 감지:
- `essence > 0` OR `meta.upgrades` 존재 OR 던전 클리어 ≥1 OR 캠페인 진행도 존재
- 위 조건 만족 시: **8 클래스 모두 자동 해제** (기존 플레이어 페널티 없음)
- 완전 신규 플레이어: **2 클래스만** (정상 진행)

이렇게 이미 게임을 즐기던 사용자가 *갑자기 클래스를 잃지 않도록* 보호. 진행도 초기화 버튼 사용 시에만 신규 상태로 리셋.

### 검증 결과

```
신규 플레이어 기본            → unlockedClasses: ['mage', 'knight'] ✓
잠긴 클래스 클릭              → 흔들림 + 선택 거부 ✓
상점 캐릭터 탭 표시           → 8 카드 (2 unlocked, 6 buy 버튼) ✓
Archer 구매 (10💎)            → 에센스 30 → 20, unlock 추가 ✓
구매 후 잠금 즉시 갱신        → 카드 컬러 + 클릭 가능 ✓
여전히 잠긴 다른 클래스       → Rogue 클릭 거부 ✓
에센스 부족 시                → 버튼 비활성 (회색) ✓
기존 플레이어 (역호환)        → 모든 8 클래스 자동 해제 ✓
```

### 게임 진행 구조 (Phase 5.7 기준)

```
1단계 (신규): 캠페인 (Mage/Knight) → 초보 던전 → 첫 에센스 5~10
2단계: 캐릭터 1~2개 해제 → 새 클래스로 캠페인 재플레이 (다른 튜토리얼!)
3단계: F2 안정 클리어 → 영구 강화 시작
4단계: F3 도달 → 한 번에 16💎 + 레어 스킬 + 깊은 강화
5단계: 모든 8 클래스 해제 + 강화 풀 → 자신만의 빌드 실험
```

각 단계가 *명확한 진행감*을 제공. 한 번에 모든 게 열리는 것 아니라 *천천히 확장*되는 구조.

### 의도적으로 미구현 (Phase 5.8+ / Phase 6+)

- **클래스별 가격 차등** — 현재 모두 10💎 균일. 일부 클래스만 더 비싸게? (디자인 결정 필요)
- **클래스 마스터리** — 같은 클래스로 N번 클리어 시 추가 패시브 잠금 해제
- **레어 스킬도 영구 잠금 해제** — 현재 던전 내에서만 유효
- **외형 스킨** — 같은 클래스 다른 외형 (커스터마이징)
- **첫 해제 축하 화면** — 6번째 캐릭터 해제 시 "컬렉션 완료!" 특별 화면

## Phase 5.8 (현재) — 클래스 마스터리

### 핵심 컨셉

캐릭터를 *해제*하는 것 + *마스터*하는 것 = 두 가지 진행감.

| 진행 층위 | 행동 | 보상 | 사이클 길이 |
|---|---|---|---|
| **단기**: 캐릭터 해제 | 에센스 모으기 (10💎) | 새 클래스 잠금 해제 | 한두 던전 |
| **장기**: 클래스 마스터 | 같은 클래스로 5회 클리어 | 패시브 ×1.5 강화 | 여러 세션 |

### 마스터리 시스템

- **카운트**: 던전 클리어 (어떤 floor에서 끝내도 1회). 캠페인 클리어는 카운트 X.
- **임계점**: **5회 클리어** = 마스터
- **저장**: `meta.classClears = { mage: 3, knight: 5, ... }` (localStorage)

### 마스터 시 효과 (각 클래스별 ×1.5 강화)

| 클래스 | 기본 패시브 | 마스터 효과 | 변경 |
|---|---|---|---|
| **Mage** | 약점 +25% | 약점 +37.5% | 수치 ×1.5 |
| **Archer** | 크리 +15% | 크리 +22.5% | 수치 ×1.5 |
| **Knight** | HP +50 | HP +75 | 수치 ×1.5 |
| **Rogue** | 첫 타격 크리 | **첫 2 타격** 크리 | 특수 처리 |
| **Priest** | 매초 +5 HP | 매초 +7.5 HP | 수치 ×1.5 |
| **Druid** | 5초마다 +5 HP | **3.3초마다** +7.5 HP | interval ÷1.5, 수치 ×1.5 |
| **Paladin** | 피해 -10% | 피해 -15% | 수치 ×1.5 |
| **Monk** | CD -20% | CD -30% | 수치 ×1.5 |

#### 특수 처리: Rogue & Druid

대부분의 패시브는 수치를 ×1.5 하면 끝이지만:

- **Rogue** (`first_hit_crit`)는 boolean 효과라 곱셈 안 됨. 대신 *조건 확장* — "첫 1회" → "첫 2회". `combat.firstHitCount` 카운터로 추적.
- **Druid** (`regen` interval)는 *짧을수록 좋음*. 그래서 5초 → 3.3초 (÷1.5)로 *주기 단축* + 수치 ×1.5.

### UI 표시

#### 클래스 선택 화면

- 카드 우상단에 **진행 배지**:
  - 0회: 표시 없음
  - 1-2회: "초보 (1/5)" 회색 배지
  - 3-4회: "⭐ 숙련 (3/5)" 회색 배지
  - 5회+: "**⭐ 마스터**" 금색 펄스 배지 + 펄스 글로우 애니메이션
- 마스터된 카드는 패시브 영역도 **황금 그라데이션**으로 강조
- 패시브 설명에 "(×1.5)" 표시

#### 영웅 카드 (전투 중)

- 패시브 라벨: 기본 "⭐ 강철 의지" → 마스터 "**⭐⭐ 강철 의지**"
- 배경 색상도 마스터 시 황금색 진하게

#### 상점 캐릭터 탭

- 클리어 카운트 표시: "ARCHER · RANGED · 클리어 3회"
- 마스터 시: 이름 옆에 **"⭐⭐ 마스터"** 황금 태그
- 패시브 이름에 **"×1.5"** 표기

### 검증 결과

```
classClears 로드/저장              → localStorage 직렬화 정상 ✓
Knight 5 클리어 → isMastered=true → 마스터 활성화 ✓
Knight 마스터 시 HP +75            → 100 + 75 = 175 (전투 시 확인) ✓
Mage 3 클리어 (아직 미마스터)      → 패시브 값 0.25 그대로 ✓
Rogue 마스터 시 first-hit-crit=2   → 첫 2 타격 크리 ✓
Druid 미마스터 interval=5초        → 5.0 ✓
영웅 카드 ⭐⭐ 표시                → "⭐⭐ 강철 의지" 확인 ✓
상점 마스터 태그 + ×1.5 표기       → 양쪽 모두 표시 ✓
카드 우상단 배지 (3/5, 마스터)      → 모두 정상 ✓
```

### 게임 진행 곡선 (Phase 5.8 기준)

신규 플레이어가 *깊은 진행감*을 느끼는 흐름:

```
🌱 1단계 (0~3 시간):
   - Mage/Knight로 캠페인 학습 (튜토리얼)
   - 첫 던전 F1 클리어 → 첫 에센스 + 첫 클래스 클리어 카운트
   - Mage 클리어 1회, 2회 누적

📈 2단계 (3~10 시간):
   - 에센스 충분 → Archer 또는 Priest 잠금 해제
   - 새 클래스로 같은 캠페인 다시 (다른 튜토리얼!)
   - 첫 클래스 마스터 도달 → 패시브 ×1.5 실감

🎯 3단계 (10~25 시간):
   - 모든 8 클래스 해제 + 영구 강화 진행
   - 자신만의 *최애 클래스* 마스터 도전
   - F3 풀 완주 안정화

🏆 4단계 (25+ 시간):
   - 8 클래스 모두 마스터 도전
   - 다양한 빌드 실험 (레어 스킬 조합)
   - 최고 기록 (총 클리어 횟수) 추적
```

### 의도적으로 미구현 (Phase 5.9+)

- **마스터+ 등급** (10회, 20회 클리어 시 추가 강화) — 현재 1 tier만
- **마스터 도전 모드** (마스터 패시브 비활성으로 클리어) — 어려운 모드
- **마스터 보너스 에센스** — 마스터된 클래스로 클리어 시 +1 에센스 추가
- **모든 클래스 마스터 시 상** — "전능자" 칭호 또는 특별 외형
- **마스터리 진행 화면** — 모든 클래스 진행도 한눈에 보는 별도 UI

## 실행

브라우저에서 `index.html` 열기. 별도 서버 불필요.

**흐름**:
1. 페이지 로드 → **클래스 선택 화면**이 먼저 등장 (8 클래스 카드)
2. 카드 클릭 → 카드 선택 (하단에 클래스 정보 + 스킬 목록 표시)
3. "▶ 전투 시작" 버튼 또는 `Enter` → 전투 화면 진입
4. 전투 화면에서 좌상단 "← 클래스 변경" 버튼 → 다시 선택 화면

**전투 화면 조작**:
- 우측 패널 드롭다운으로 자산 선택 (적/배경 등). 적을 바꾸면 전투 새로 시작
- 영문 알파벳 타이핑 → 스킬 단어 매칭 시 발동 + 적 피격
- `Backspace` 한 글자 지움
- `Esc` 또는 `Space` 전체 클리어
- "Random combo" 버튼으로 무작위 자산 + 새 전투
- 승리/패배 오버레이에서 `Space`/`Enter` 또는 클릭으로 진행
